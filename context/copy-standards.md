# RemotePass — Copy Standards

## Voice
Clear, warm, direct. Like a knowledgeable colleague, not a corporate tool.

## Tone rules
- No jargon
- No passive voice
- No filler words: "seamlessly", "powerful", "robust", "leverage", "streamline"
- Speak to what the user is doing, not what the product does

## Placeholder text
- Free text / number inputs: `e.g. [realistic example]` — e.g. "e.g. John Smith"
- Selects / dropdowns: `Select [noun]` — e.g. "Select a country"
- Never repeat the field label in the placeholder
- Never use "Add value", "Enter text", or "Type here"

## Error messages
- Say what happened and what to do next
- Don't blame the user
- Be specific: "Email already in use" not "Something went wrong"

## Empty states
- Acknowledge the empty state directly
- Give a clear CTA if there's an action to take
- Don't use "No data found" or "Nothing here yet"

## Confirmation dialogs
- Title: verb + noun — e.g. "Delete Contract?"
- Body: say what will happen (consequences), not just repeat the action
- CTA: mirrors the title verb — e.g. "Delete", not "Confirm"

## Character limit guidance (approximate)
- Button labels: 1–3 words, max 20 chars
- Tooltip: max 80 chars
- Toast/snackbar: max 60 chars
- Modal title: max 50 chars
- Modal body: max 150 chars
- Empty state headline: max 40 chars
- Empty state body: max 100 chars
